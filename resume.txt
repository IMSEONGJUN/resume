# 자유롭게 자신을 소개해주세요.

<개인 사업의 시작과 끝>

홈쇼핑 회사에서 MD로 근무하다가 퇴사 후 개인 사업을 시작하였습니다. 회사를 운영하면서 단순 반복 작업이 늘어나기 시작했고, 나중에는 제대로된 자동화 프로그램이 필요하게 되었습니다. 되도록 저의 사업에 딱 맞는 프로그램을 직접 개발하고 싶었지만, 개발에 대한 지식도 없었고, 영세한 사업 규모에서 그런 부분들을 제대로 챙기지 못하였습니다. 결국 재정적인 상황으로 2018년 9월에 사업을 접게 되었습니다.

<또 다른 시작>

그 이후, 사업의 실패 원인을 분석하면서 결국 업무 자동화 및 효율화를 꾀할 수 있는 프로그램의 부재가 컸다고 봤고, 이전부터 관심이 있었던 업무 자동화 프로그램 개발을 본격적으로 해보기 위해 개발 공부를 시작하게 되었습니다. 컴퓨터 공학을 전공하고 있는 친척 동생의 도움으로 전공 관력 서적들을 빌려 독학하고, C/C++ 언어를 배우며 간단한 프로그램들을 만들 수 있게 되었습니다.
 
TCP Socket를 사용하여 1:1 대전 오목 게임을 제작하면서 개발에 재미를 붙이고, 작년 11월부터는 제가 사용하는 아이폰에서 동작하는 앱을 만들어보고 싶다는 생각에 오전 10시부터 밤 10시까지 일정으로 주 5일간 참여하는 iOS 개발 부트 캠프에 등록하여 지금 현재 저의 목표가 된 iOS 개발자로의 첫 발을 내디딜 수 있었습니다.

<iOS 개발, 열정을 일으키다>

iOS 개발은 C++보다 훨씬 더 재미있었습니다. 지난해 11월에 시작한 iOS 개발 공부에 푹 빠져서 지금까지 하루 평균 10시간씩 약 4개월의 시간을 보내왔습니다. 살면서 이렇게까지 무언가에 열정적으로 매달려 본 적이 없었습니다. 코딩을 하는 것, 관련된 새로운 것을 배우는 것, 구현에 성공하는 것, 이러한 것들이 저에게 엄청난 희열을 가져다주었습니다. 이제는 취미이자 직업이 되었으면 하는 개발에서 늦게나마 저의 적성을 찾았다는 점에서 너무 행복합니다.
 
이전에는 없던 인생의 좌우명도 생겼습니다. 그것은 ‘인생은 속도보다 방향이 중요하다’입니다. 대학을 졸업하고 유통업에서 회사 생활을 하고, 퇴사하여 개인 사업을 하며 노력했지만, 저는 또다시 새로운 분야에 도전하고 있습니다. 남들보다 늦은 나이에 시작한 일이기에 두려움이 앞섭니다. 하지만 새롭게 시작한 개발  공부가 너무 재미있고 스스로 상당히 즐기고 있어서 이제서야 인생의 방향을 제대로 정했다고 생각하고 있습니다. 포기하지 않고 내가 정한 방향으로 나아가다 보면 반드시 원하는 인생에 조금 더 가까워질 수 있을 것이라고 스스로 마음을 다잡습니다.


# 본인이 수행한 대표적인 프로젝트를 한 개 이상 적어주시고, 프로젝트에서 어떤 기술과 방식을 활용하여 개발했는지 설명해 주세요.

AppStore:  https://apps.apple.com/kr/app/github-followers/id1497318994 
Git Repo: https://github.com/IMSEONGJUN/GitHubFollowers

<GitHub 사용자 정보 검색을 및 저장을 위한 GitHub Followers 프로젝트>

개인적으로 진행한 프로젝트로 깃헙 아이디를 입력하면 해당 아이디의 Follower 또는 Following 데이터를 github API를 이용하여 불러와 CollectionView로 구성된 UI에 리스트를 보여주는 기능을 구현하였습니다. API 비동기 작업은 별도의 Singleton 클래스로 구현하였습니다. ViewController를 커스텀하여 UI구성을 위한 Child ViewController로 활용하였습니다. View의 상속을 통해 코드양을 줄이고 재사용성이 높아지도록 노력하였습니다. 
프로젝트 후반부에 연결 접점이 없는 ViewController 객체간의 Communication을 해야 했는데. 프로젝트 당시 NotificationCenter를 활용하는 방법을 배우지 못해서 구현에 어려움이 있었습니다. 그래서 Delegation을 활용하여 중간 다리 역할을 하는 객체를 한번 거쳐서 delegate 메소드가 구현된 객체에 호출될 수 있도록 하여 구현하였습니다. 프로젝트 완성 후에 Appstore에 배포하였습니다. 

<내 옷장 관리를 위한 앱 MyCloset>

부트캠프에서 2인 1조로 진행한 해커톤에서 최신 트렌드의 패션스타일을 이미지로 소개하고, 사용자의 옷들을 카테고리별로 관리하여 코디세트를 만드는 프로젝트를 진행하였습니다. 저는 프로젝트 대부분의 UI 구성을 담당하였고 Google Firebase를 활용하여 저장된 코디세트 이미지들의 불러와서 보여주는 기능, UIView 객체를 UIImage 객체로 변환해주는 기능을 담당하였습니다. 메인 UI 구성에 Pinterest 스타일의 CollectionView Layout를 적용하기 위해 UICollectionViewFlowLayout 클래스를 서브클래싱하여 별도의 커스텀 FlowLayout 클래스를 구현하였습니다. 각 Cell 마다 좌표와 크기를 지정해주기 위해 UICollectionViewLayoutAttributes 타입의 배열객체 생성하고 override한 prepare() 메소드에서 각각의 Cell 속성 값들을 구성하였습니다.

<1:1 네트워크 대전 오목 게임>

C++로 수행한 Windows용 네트워크 오목 게임 프로젝트입니다. Win32 API를 사용하여 화면에 오목 화면을 그리고, WinSock을 사용하여 1:1로 대전이 가능하도록 만들었습니다. 사용한 프로토콜은 TCP이고 이벤트 제어는 IOCP 방식을 사용하였습니다. Packet 구성을 직접하여 어느 사용자가 어느 위치에 어떤 색의 바둑돌을 놓았는지 서로 통신을 하였고, 별도의 서버없이 P2P로 Packet을 주고 받았습니다.
